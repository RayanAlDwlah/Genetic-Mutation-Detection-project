📌 1. Project Overview

This project builds an end-to-end ML pipeline to classify human missense single-nucleotide variants (SNVs). A missense variant is a single DNA base change that alters one amino acid in a protein. When such changes disrupt critical protein function, they can cause genetic diseases.

🎯 Task

Binary classification: Is a given missense variant Pathogenic (disease-causing) or Benign (harmless)?

📊 Data

Labels from ClinVar. Features from gnomAD (population frequencies) and dbNSFP (conservation + physicochemical properties).

🤖 Models

Phase 1: XGBoost gradient boosting (complete). Phase 2: 1D CNN and ESM-2 protein language model (after midterm).

ℹ️
Why missense variants? They are the most common type of disease-causing point mutation. Databases like ClinVar contain hundreds of thousands of expert-curated examples, making them ideal for supervised classification.

🐍 8. Source Code Guide (src/)

src/utils.py

What it does: Shared helpers used across all pipeline scripts.

  • normalize_chromosome(value) — converts any chromosome format (e.g., chr1, 23) to standard form (1, X)
  • resolve_path(repo_root, path) — makes relative paths absolute against the project root
  • load_yaml_config(path) — loads configs/config.yaml
  • require_file(path) — raises a clear error if a required file is missing

src/clinvar_cleaning.py

What it does: 5-step pipeline to produce a clean, labeled ClinVar dataset.

Steps: Filter SNVs → keep 4 clinical labels → assign binary labels (0/1) → filter by review stars → normalize chromosomes → deduplicate → remove conflicting labels.

Run: python -m src.clinvar_cleaning --config configs/config.yaml

src/gnomad_extraction.py

What it does: Extracts allele frequencies (AF, AC, AN, AF_popmax, log_AF, is_common) from the gnomAD VCF.

Special feature: Uses a StreamingParquetSink to process the 500 GB+ VCF in chunks, keeping memory usage constant regardless of file size.

Run: python -m src.gnomad_extraction --input data/raw/gnomad/...vcf.bgz --clinvar-variants data/intermediate/clinvar_labeled_clean.parquet

src/dbnsfp_extraction.py

What it does: Reads the 100 GB+ dbNSFP table and extracts conservation scores, amino acid properties, and BLOSUM62/Grantham substitution scores for each variant.

Important: Automatically excludes all ClinVar-derived meta-predictors (REVEL, ClinPred, etc.) to prevent circular evaluation.

src/evaluation.py

What it does: Computes all classification metrics from predicted probabilities.

  • compute_classification_metrics(y_true, y_prob, threshold) — returns ROC-AUC, PR-AUC, F1, MCC, precision, recall, Brier loss
  • select_best_threshold(y_true, y_prob) — grid search over [0.20, 0.80] on the validation set to maximize F1

src/models/xgboost_model.py

What it does: Configures and runs XGBoost hyperparameter tuning.

Strategy: 14 trials — trial 0 uses conservative defaults, trials 1-13 sample from random ranges. Best model selected by: 0.65 × ROC-AUC + 0.35 × PR-AUC.

Key class: XGBTuningConfig(n_trials=14, n_estimators=2500, early_stopping_rounds=80)

🔬 10. Key Design Decisions

Gene-Level Split (Most Critical)

In a random split, variants from the same gene appear in both training and test sets. Since pathogenicity is partly gene-specific (e.g., BRCA1 variants tend to be pathogenic), a random-split model would learn gene identity → label instead of functional features → label. This would inflate ROC-AUC by 5–15%.

Solution: Use GroupShuffleSplit with gene as the group key. Every gene is assigned to exactly one split. Zero gene overlap is verified programmatically in Notebook 06.

Circularity Protection

Meta-predictors like REVEL and ClinPred are trained on ClinVar data themselves. Including them as features would mean predicting ClinVar labels using models already optimized for ClinVar labels — performance numbers would be inflated and not reproducible on external benchmarks.

Solution: Explicitly exclude all 8 ClinVar-derived composite scores from the feature set at the extraction stage.

Two Dataset Versions (Ablation)

Balanced: 283K variants, review stars ≥ 1 (includes single-submitter entries), missing values imputed with training-set median. More data → better generalization.

Strict: 37K variants, review stars ≥ 2 (multiple submitters or expert panel), no imputation. Higher-confidence labels → cleaner signal.

The ablation study in Notebook 08 evaluates the same model on both test sets to quantify the data quantity vs. quality trade-off.

Threshold Optimization

Default classifiers use a 0.5 decision threshold. In clinical genetics, false negatives (missed disease-causing variants) carry higher cost than false positives (unnecessary follow-up). Solution: Grid search over [0.20, 0.80] on the validation set only, maximizing F1 score. The best threshold (0.445) is then applied to the held-out test set without further adjustment.
