📌 1. Project Overview

This project builds an end-to-end ML pipeline to classify human missense single-nucleotide variants (SNVs). A missense variant is a single DNA base change that alters one amino acid in a protein. When such changes disrupt critical protein function, they can cause genetic diseases.

🎯 Task

Binary classification: Is a given missense variant Pathogenic (causes disease) or Benign (harmless)?

📊 Data

Labels from ClinVar (clinical database). Features from gnomAD (population frequencies) and dbNSFP (conservation scores).

🤖 Models

Phase 1: XGBoost (done). Phase 2: 1D CNN and ESM-2 protein language model (after midterm).

ℹ️
Why missense variants? They are the most common type of disease-causing mutation and are directly interpretable from protein structure and function. Databases like ClinVar have thousands of curated examples.

🐍 8. Source Code Guide (src/)

src/utils.py

What it does: Shared helpers used by all scripts.

  • normalize_chromosome(value) — converts any chromosome format (e.g., chr1, 23) to standard form (1, X)
  • resolve_path(repo_root, path) — makes relative paths absolute against the project root
  • load_yaml_config(path) — loads configs/config.yaml

src/clinvar_cleaning.py

What it does: 10-step pipeline to go from raw ClinVar to a clean, labeled dataset.

Key steps: Filter SNVs → keep 4 labels → assign binary labels → filter by review stars → normalize chromosomes → deduplicate → remove conflicting labels.

Run: python -m src.clinvar_cleaning --config configs/config.yaml

src/gnomad_extraction.py

What it does: Extracts allele frequencies (AF, AC, AN, AF_popmax) from gnomAD.

Special feature: Uses a StreamingParquetSink to process the 500GB+ VCF file in chunks without loading it into memory. Falls back to gnomAD API for Colab environments.

Run: python -m src.gnomad_extraction --input data/raw/gnomad/...vcf.bgz --clinvar-variants data/intermediate/clinvar_labeled_clean.parquet

src/dbnsfp_extraction.py

What it does: Reads the 100GB+ dbNSFP table and extracts conservation scores, amino acid properties, and BLOSUM62/Grantham substitution scores for each variant.

Important: Automatically excludes ClinVar-derived predictors (REVEL, ClinPred, etc.) to prevent circularity.

src/evaluation.py

What it does: Computes all classification metrics from predicted probabilities.

Key functions:

  • compute_classification_metrics(y_true, y_prob, threshold=0.5) — returns ROC-AUC, PR-AUC, F1, MCC, precision, recall, confusion matrix, Brier loss
  • select_best_threshold(y_true, y_prob) — grid search over [0.2, 0.8] to find the threshold that maximizes F1 on the validation set

src/models/xgboost_model.py

What it does: Configures and runs XGBoost hyperparameter tuning.

Strategy: 14 trials — trial 0 uses a conservative baseline, trials 1-13 sample from randomized ranges. Best model selected by: 0.65 × ROC-AUC + 0.35 × PR-AUC.

Key class: XGBTuningConfig(n_trials=14, n_estimators=2500, early_stopping_rounds=80)

🔬 10. Key Design Decisions

Gene-Level Split (Most Important)

In a random split, the same gene's variants appear in both training and test sets. Since pathogenicity is partly gene-specific (e.g., BRCA1 variants tend to be pathogenic), the model could learn gene identity → label instead of functional features → label.

Solution: Use GroupShuffleSplit on gene as the group key. Every gene is assigned to exactly one split. Zero overlap is verified programmatically.

Circularity Protection

Meta-predictors like REVEL and ClinPred are trained on ClinVar data. Including them as features would mean we're predicting ClinVar labels using models already fitted on ClinVar labels. Performance numbers would be inflated and meaningless for external validation.

Solution: Explicitly exclude all ClinVar-derived composite scores from our feature set.

Two Dataset Versions

Balanced: 283K variants, review stars ≥ 1 (includes single-submitter entries), median-imputed missing values. More data = better generalization.

Strict: 37K variants, review stars ≥ 2 (multiple submitters or expert panel), no imputation. Higher confidence labels = cleaner signal.

The ablation study in notebook 08 evaluates the same model on both test sets to quantify this trade-off.

Threshold Tuning

By default, a classifier uses 0.5 as the decision threshold. But our classes have different clinical costs (false negatives = missed disease are worse than false positives = unnecessary follow-up). Solution: Grid search over [0.2, 0.8] on the validation set only, optimizing F1 score. The best threshold (0.445) is then applied to the test set.
